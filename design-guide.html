UI & motion design system

Serious, readable, and calm. Motion supports comprehension — it never competes with content. This page documents tokens and behaviour; the canonical written spec is in the repository root as DESIGN_GUIDE.md.

Written 10 April 2026 · Last updated 21 April 2026 · Implementation: css/theme.css · css/motion.css · js/motion.js

Themes

Shared typography — IBM Plex Sans (UI and body) and IBM Plex Mono (labels, data, navigation) across sector and institutional reports.

Light default, optional dark — tokens live in css/theme.css; html[data-br-theme="dark"] flips the palette. Preference is stored in localStorage via the masthead toggle.

Institutional layout — same fonts and tokens; cover and section structure stay editorial (large cover title, stat row, layer tables) with colours driven by the same theme variables.

Motion & transitions

All motion respects prefers-reduced-motion. Users who disable animation get static layouts with no scroll-driven opacity tricks.

TokenValueUse
--br-ease-outcubic-bezier(0.22,1,0.36,1)Page enter, reveals
--br-duration-slow0.6sScroll reveal
--br-duration-fast0.2sChrome / masthead
  • Page enter — short translate on nav, main, .page, or .cover after load (no parent opacity animation).
  • Cross-page — when supported, document.startViewTransition wraps same-origin navigation to *.html links.
  • Scroll reveal — add data-reveal; motion.js toggles .br-is-visible once the block enters the viewport.
  • Optional delay per block — set style="--br-reveal-delay: 0.1s" on a [data-reveal] node.

Layout & chrome

  • Masthead — fixed 44px bar (.br-masthead) on every page; document body uses .br-masthead-spacer to offset content.
  • Consumer in-page nav — second sticky row under masthead (body.has-local-nav nav { top: 44px }).
  • Focus — visible focus rings on links and controls; gold on dark theme, dark gold on light masthead.
