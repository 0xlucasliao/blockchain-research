The Gap: Strong Infrastructure, Underdeveloped Payment Utility
Thin card rails. Many high-throughput EVM networks lack native, network-aligned debit-card programs; users depend on generic crypto cards with little chain-specific distribution or UX.
Uneven PSP coverage. Some ecosystems (e.g. Base with Stripe, Solana with Shopify/Circle) have major acquirer paths; others still have no first-class PSP or checkout integration despite high stablecoin throughput.
Agentic tooling early. Few places offer an end-to-end SDK plus virtual-card and policy primitives for non-custodial, payment-capable agents on a given chain.
MPP / UCP still sparse. Machine Payment Protocol and Universal Commerce Protocol implementations are not yet ubiquitous; early movers can contribute reference code and extensions.
Checkout vs P2P. High stablecoin transfer counts rarely translate into broad merchant checkout acceptance; off-exchange retail often still flows through a small set of closed-loop apps.
Non-technical users. Hosted agents and one-click deployment are not yet standard; large address counts do not by themselves onboard mainstream users to agentic wallets.

Infrastructure & Tools to Build

Infrastructure gaps to close for each stakeholder — chain-agnostic building blocks

Reference: Base
Base Pay SDK + Base Account SDK
  • Two-line payment integration: import { pay } from '@base-org/account/pay'
  • Drop-in UI components: BasePayButton, SignInWithBaseButton
  • Works without wallet connection — operates independently via Base Account SDK
  • Supports Vue, Preact, Svelte alongside React
What we learn
Make the first integration under 5 lines of code. UI components reduce merchant implementation friction to near zero.
Reference: Solana
Solana Commerce Kit + Solana Pay
  • Modular packages: kit (all-in-one), react, sdk, headless, connector, solana-pay
  • PaymentButton component — drop-in with merchant wallet address and amount
  • QR code payments via Solana Pay protocol (open standard)
  • Shopify integration live — millions of merchants via one app install
  • Partnership model: Circle (USDC), Phantom (wallet), Checkout.com (PSP)
What we learn
Build a payment protocol first, then wrap it in SDKs. A Shopify plugin multiplies reach instantly — one integration unlocks millions of merchants.
Wallet Providers
Wallet payment integration SDK
What
An SDK that lets any crypto wallet (Trust Wallet, WalletConnect-enabled apps, MetaMask) integrate on-chain payment acceptance, live asset price feeds, and balance management in a few lines of code.
Why
Wallets are the customer-facing entry point — without wallet integration, users cannot initiate Path A or B flows from software they already trust.
Impact
Each wallet integration is distribution: payment features reach that wallet’s entire user base from day one.

Issuers (Path A)
Card issuance SDK (crypto-funded)
What
A framework for issuers (Marqeta, Galileo, RedotPay) to issue debit and virtual cards funded from on-chain balances, with BIN integration, JIT crypto-to-fiat swap, and spend policy enforcement.
Why
Card issuance is how crypto-funded users and agents reach the full Visa/Mastercard merchant universe without merchants changing checkout.
Impact
One issuer integration unlocks access to every merchant that accepts Visa/MC globally — the fastest path to broad merchant reach.

PSPs / Acquirers (Path B)
Acquirer payment SDK
What
An SDK enabling PSPs (Stripe, Adyen, Fiserv) to accept major digital assets and chain-native stablecoins as a payment method, with FX routing and crypto-to-fiat settlement to merchant accounts.
Why
PSPs control merchant payment acceptance — one PSP integration can expose crypto settlement to many merchants without each merchant building custom chain support.
Impact
Mirrors what newer chains have pursued with Tier-1 PSPs: crypto becomes a selectable payment method inside existing dashboards.

Agents (Path C)
Agent Wallet & Card SDK
What
A TypeScript/Python SDK for AI agents to create and manage virtual cards, set and enforce spend policies, delegate permissions, and route payments across card or on-chain rails automatically.
Why
Today, virtually no non-technical user has deployed an AI agent because there is no simple payment primitive — this SDK is the missing layer that makes agents commercially useful.
Impact
Every deployed agent with spend authority becomes a recurring transaction generator — a funded agent with a virtual card is effectively a daily active user of whichever rail settles the spend.

Shared Infrastructure
Liquidity & Swap Layer
What
A chain-agnostic liquidity integration layer that gives issuers, acquirers, and PSPs access to guaranteed-rate crypto-to-fiat swaps and real-time off-ramp execution.
Why
Every adapter path (A, B, C) requires a crypto-to-fiat conversion step — without a reliable, low-slippage liquidity layer, no integration is commercially viable.
Impact
A shared liquidity layer means each new PSP or issuer integration gets instant swap capability without negotiating their own liquidity deals.

Protocols
MPP SDK + UCP extension
What
Ship an MPP SDK aligned with the Machine Payment Protocol for agent-to-processor messaging, and contribute a Universal Commerce Protocol (UCP) fork or extension that adds a blockchain payment leg — an early on-chain UCP shape others can interoperate with.
Why
MPP is how agents express payment intent to processors; without it, agents stay outside emerging agentic commerce standards. UCP’s on-chain story is still thin — reference implementations help the whole field.
Impact
Standard-level contributions outlast any single PSP deal: documentation and working code anchor ecosystems in the next wave of agentic checkout.

Go-to-Market Strategy

Who we target, in what order, and with what narrative

Priority 1: Smaller PSPs
Start with smaller PSPs and acquirers — move fast
  • Why smaller first: Large PSPs (Stripe, Adyen) are effectively locked to Base and Polygon via existing agreements. Smaller PSPs move faster, have fewer crypto commitments, and are hungry for differentiation.
  • Goal: Use smaller PSPs to validate an acquirer payment SDK, build reference integrations, and generate a working portfolio before approaching Tier 1.
  • Campaign hook: "Be the first PSP in your market to settle major digital assets — we provide the SDK, liquidity partners, and co-marketing."
Smaller regional PSPs
Mid-market acquirers

Priority 2: Tier 1 PSPs
Approach Stripe, Adyen, Fiserv with proven integrations
  • Stripe: Already integrated several L2s — pitch a differentiated chain or bundle of assets with a ready SDK and live pilot volume from smaller PSPs first.
  • Adyen + Fiserv: Neither has meaningful crypto acceptance today — first-mover opportunity with enterprise merchants neither Stripe nor Base currently serves well.
  • Leverage: Come with live integration data, transaction volume, and a merchant waitlist. Tier 1 PSPs respond to demonstrated demand, not pitches.

Priority 3: Wallets + Issuers
Enable Path A — crypto debit card issuance
  • Wallet partners: Trust Wallet, WalletConnect-compatible wallets, MetaMask, and any wallet with issuer connections. Confirm product roadmap alignment on card issuance.
  • Issuer partners: RedotPay (most accessible, immediate), Marqeta, Galileo for enterprise-grade card programs. Need BIN number access and JIT swap capability.
  • Agent cards: Every agent deployed via launchpad partners (NodeOps) gets a virtual card — the agent card program is the highest-leverage card issuance volume driver.

Parallel: Agent Launchpads
Unlock the non-technical majority — free agent deployment
  • The gap: Despite agentic hype, few non-technical users run their own funded agents. Removing deployment friction is a cross-chain opportunity, not a single-network exclusive.
  • Model: Partner with agent launchpad platforms to offer one-click agent deployment tied to wallet balances; infrastructure partners (e.g. NodeOps-style hosts) can lower backend cost.
  • Flywheel: More agents deployed → more virtual cards → more measurable payment volume → stronger leverage in PSP and issuer conversations for whichever ecosystems fund the loop.

Campaign Narratives by Stakeholder
For Wallets
"Let your users spend their crypto anywhere, today."
Wallets integrate a payment SDK to unlock QR flows, card-linked spending, and live price management. Users get real-world utility for on-chain balances without leaving the wallet app.
For Issuers
"Issue the card of the crypto economy — with zero liquidity risk."
Ecosystem builders supply the card-spec SDK, liquidity for JIT swaps, and a user pipeline ready to activate cards. Issuers keep the BIN relationship; builders abstract chain complexity.
For PSPs / Acquirers
"Add crypto acceptance in a weekend — your merchants won't notice a thing."
An acquirer SDK slots into existing checkout flows. Merchants get a new settlement option with minimal UX change. PSPs earn on spread and services; chain details stay behind PSP APIs.
